Blog: Smoke-testing an LLM agent with a Claude Code skill

MailPilot is an agent-driven CRM where the business logic lives inside a Pydantic AI agent, so I test it in three layers: ruff + basedpyright and pytest verify the machinery, and a Claude Code smoke-test skill verifies the agent against real Gmail and a real knowledge base. Claude Code is the runner, the assertions live in deterministic gates plus one structured-JSON LLM judgment, and every recurrence-class failure auto-files into SPEC.md via /sdd:spec.

The standard testing pyramid inverts when an LLM sits at the core: the agent can pass every mocked test and still fabricate vendor specs that don't exist. So I run three layers, each catching what the layer below cannot.

- ruff + basedpyright — type errors, import cycles, shape drift in three seconds. Pytest cannot replace this layer.
- pytest — real Postgres, mocked HTTP boundaries, structured Logfire span assertions. Verifies the wiring.
- A Claude Code smoke-test skill — real Gmail, real Drive, real model. Verifies that the agent works.

The smoke-test runner is Claude Code itself. Gates are mostly deterministic; the one LLM-judged gate forces structured JSON with an unsupported_claims array — the anti-sycophancy lever. Critical bug failures auto-file as SPEC entries via /sdd:spec from the same Claude Code session, with the failed verdict JSON inline. No copy-paste between tools.

The downside is honest: roughly seven minutes of wall clock per run plus an LLM round-trip per step. It is not a CI gate. It is a pre-commit ritual after a non-trivial change to the agent surface — the way you would run integration tests against staging, when the diff justifies it, not on every push.
